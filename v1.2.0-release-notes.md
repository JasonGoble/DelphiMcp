## ⚠️ Deprecation & Upgrade Notice

This release marks the deprecation of library-specific MCP tools in preparation for removal in v1.3. All users should plan to migrate to unified tools.

### What's Changing

**Library-specific tools are now deprecated:**
- `search_rtl` → Migrate to `search_delphi_source` with `library="rtl"`
- `lookup_rtl_class` → Migrate to `lookup_delphi_class` with `library="rtl"`
- `search_devexpress` → Migrate to `search_delphi_source` with `library="devexpress"`
- `lookup_devexpress_class` → Migrate to `lookup_delphi_class` with `library="devexpress"`

### Key Dates

- **v1.1 (released)**: Unified tools available; old tools functional
- **v1.2 (now)**: Old tools emit `[Obsolete]` compiler warnings; profiles optional
- **v1.3 (next)**: Old tools removed entirely (semver major bump)

### What You Need to Do

**If using v1.1 or earlier:**
1. Read the v1.2 Breaking Changes section in [README.md](https://github.com/JasonGoble/DelphiMcp/blob/main/README.md#v12-breaking-changes--deprecation)
2. Update your code to use unified tools (examples provided in README)
3. Configure ClientAccess profiles with your machine's default libraries (optional but recommended)
4. Test your integration with v1.2 to see deprecation warnings
5. Plan to upgrade to v1.3+ once migration is complete

**If already using v1.1 unified tools:**
- No action required for v1.2
- Prepare for v1.3 removal of old tools

### Migration Examples

**v1.0 code** (library-specific):
```csharp
var result = await mcp.CallTool("search_rtl", new { query = "TStringList", topK = 5 });
```

**v1.1+ code** (unified):
```csharp
var result = await mcp.CallTool("search_delphi_source", new { 
    query = "TStringList", 
    library = "rtl", 
    topK = 5 
});
```

Or with profile defaults:
```csharp
var result = await mcp.CallTool("search_delphi_source", new { 
    query = "TStringList",
    topK = 5  // searches profile's DefaultScopes
});
```
